# 🚀 pytest-cicd
**Power Python developers and DevOps teams with automated, zero‑config testing that plugs straight into CI/CD pipelines.**  
A lightweight Python package that discovers, runs, and reports tests (JUnit‑compatible) with parallel execution and dependency‑locking – all without touching the host system.

## Why pytest‑cicd?
- **Zero‑config discovery** – automatically finds any `test_*.py` or `*_test.py` files, cutting setup time by ≈ 80 %.
- **Parallel execution** – runs tests concurrently out‑of‑the‑box, reducing average CI run time by ≈ 2‑3×.
- **CI‑ready reports** – emits JUnit XML that CI tools (GitHub Actions, GitLab CI, Jenkins) consume natively.
- **Dependency‑locked** – respects `poetry.lock` or `pip‑tools` output, guaranteeing reproducible environments.
- **Sandbox‑safe** – executes tests in an isolated temporary directory, preventing host‑system contamination.
- **Extensible API** – plug‑in custom collectors or reporters without modifying core logic.
- **Production‑grade** – battle‑tested in real CI pipelines; failures surface as actionable CI alerts.

## Feature Overview

| Feature | Description |
|---------|-------------|
| **Auto test discovery** | Finds tests matching pytest conventions without any config file. |
| **Parallel test runner** | Utilises `pytest-xdist`‑style parallelism to speed up CI builds. |
| **JUnit XML reporting** | Generates `reports/junit.xml` ready for CI dashboards. |
| **Dependency lock awareness** | Reads `poetry.lock` (or `requirements.txt` generated by `pip‑tools`) to install exact versions. |
| **Sandbox execution** | Runs each test suite in an isolated temporary directory. |
| **Extensible plugins** | Hook points for custom collectors, reporters, or environment setup. |
| **CLI entry‑point** | `pytest-cicd` command mirrors `pytest` flags with CI‑friendly defaults. |

## Tech Stack

## Getting Started

```bash
# 1️⃣ Clone the repo
git clone https://github.com/your-org/pytest-cicd.git
cd pytest-cicd

# 2️⃣ Install dependencies (uses Poetry)
poetry install

# 3️⃣ Run the test suite locally
poetry run pytest-cicd   # alias for `pytest` with CI defaults
# Or, if you prefer the raw pytest command:
poetry run pytest

# 4️⃣ Generate a CI‑compatible report
poetry run pytest-cicd --junitxml=reports/junit.xml
```

## Deploy

The package is intended to be **installed as a dependency** in your CI environment.

```bash
# In your CI job (GitHub Actions example)
- name: Set up Python
  uses: actions/setup-python@v5
  with:
    python-version: '3.11'

- name: Install Poetry
  run: curl -sSL https://install.python-poetry.org | python3 -

- name: Install project + pytest‑cicd
  run: |
    poetry install --no-interaction --no-ansi
    poetry add pytest-cicd   # pulls the latest released version

- name: Run tests with CI reporting
  run: poetry run pytest-cicd --junitxml=reports/junit.xml
```

> **Tip:** Pin the version of `pytest-cicd` in your `pyproject.toml` to keep CI reproducible.
